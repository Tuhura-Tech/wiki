Imagine getting search results but they're all mixed up. Sorting puts things in order, and limiting shows you just the top results. Let's learn to organize our data.

## ORDER BY - Sorting Results

`ORDER BY` arranges your results in order - like organizing your bookshelf or lining up by height!

### Basic Sorting (Ascending = Smallest to Largest)

```sql
SELECT * FROM Customers
ORDER BY age;
```

**Output:**
| customer_id | first_name | last_name | age | country |
|-------------|------------|-----------|-----|---------|
| 2           | Robert     | Luna      | 22  | USA     |
| 3           | David      | Robinson  | 22  | UK      |
| 4           | John       | Reinhardt | 25  | UK      |
| 5           | Betty      | Doe       | 28  | UAE     |
| 1           | John       | Doe       | 31  | USA     |

Ages go from youngest (22) to oldest (31)!

### DESC - Reverse Order (Descending = Largest to Smallest)

```sql
SELECT first_name, last_name, age FROM Customers
ORDER BY age DESC;
```

**Output:**
| first_name | last_name | age |
|------------|-----------|-----|
| John       | Doe       | 31  |
| Betty      | Doe       | 28  |
| John       | Reinhardt | 25  |
| Robert     | Luna      | 22  |
| David      | Robinson  | 22  |

Now oldest first!

## Multiple Column Sorting

You can sort by multiple columns! It's like organizing books by genre first, then by author!

### Sort by Game, Then by Score
```sql
SELECT username, game, score FROM GameScores
ORDER BY game, score DESC;
```

**Output:**
| username     | game       | score |
|--------------|------------|-------|
| PixelMaster  | Fortnite   | 91000 |
| CoolKid99    | Fortnite   | 89000 |

First sorted by game name, then within each game, highest scores first!

## Combining Everything!

Let's use WHERE, ORDER BY, and LIMIT together!

### Top 2 Minecraft Players by Score
```sql
SELECT username, score, level FROM GameScores
WHERE game = 'Minecraft'
ORDER BY score DESC
LIMIT 2;
```

**Output:**
| username     | score | level |

| DragonSlayer | 67000 | 35    |

SELECT name, species, age, weight_kg FROM ZooAnimals

## Pro Tips

### Tip 1: ORDER BY Column Position
You can use numbers instead of column names!
```sql
SELECT name, score FROM GameScores
ORDER BY 2 DESC;  -- Sort by 2nd column (score)
```

### Tip 2: LIMIT with OFFSET
Skip the first few results!
```sql
-- Skip top 3, show next 2
SELECT username, score FROM GameScores
ORDER BY score DESC
LIMIT 2 OFFSET 3;
```

**Output:**
| username     | score |
|--------------|-------|
| DragonSlayer | 67000 |
| NinjaWarrior | 52000 |

### Tip 3: ASC is Default
You don't need to write ASC, it's automatic!
```sql
ORDER BY age      -- Same as ORDER BY age ASC
ORDER BY age ASC  -- Explicit ascending
```

:::tip[Think Like a Librarian!]
Sorting is like organizing a library:
- **ORDER BY name** = Alphabetical order
- **ORDER BY age DESC** = Newest to oldest
- **LIMIT 10** = Show only the top 10
- Combine them to find exactly what you need!
:::
