The `SELECT` statement is like asking the database a question. Let's learn how to get exactly the information we need.

## The Basic SELECT

The simplest way to ask for data is to select everything from a table:

```sql
SELECT * FROM Customers;
```

The `*` means "give me ALL columns!" It's like saying "I want to see EVERYTHING!"

**Try it online:** Open [Programiz SQL Compiler](https://www.programiz.com/sql/online-compiler/) and run this!

**Output:**
| customer_id | first_name | last_name | age | country |
|-------------|------------|-----------|-----|---------|
| 1           | John       | Doe       | 31  | USA     |
| 2           | Robert     | Luna      | 22  | USA     |
| 3           | David      | Robinson  | 22  | UK      |
| 4           | John       | Reinhardt | 25  | UK      |
| 5           | Betty      | Doe       | 28  | UAE     |

## Choosing Specific Columns

Sometimes you don't need everything! You can choose which columns to see:

```sql
SELECT first_name, last_name FROM Customers;
```

This is like saying "I only want to know their names, not their age or country!"

**Output:**
| first_name | last_name |
|------------|-----------|
| John       | Doe       |
| Robert     | Luna      |
| David      | Robinson  |
| John       | Reinhardt |
| Betty      | Doe       |

**Pick different columns:**
```sql
SELECT first_name, age, country FROM Customers;
```

**Output:**
| first_name | age | country |
|------------|-----|---------|
| John       | 31  | USA     |
| Robert     | 22  | USA     |
| David      | 22  | UK      |
| John       | 25  | UK      |
| Betty      | 28  | UAE     |

## Let's Create Our Own Table!

Let's make a fun table about ice cream flavors! Copy this into the online editor:

```sql

  flavor_name TEXT,
  color TEXT,
  scoops_sold INTEGER,

INSERT INTO IceCream VALUES
```

## Doing Math in SELECT!

You can do calculations right in your SELECT statement! It's like having a calculator inside your database!

### Add Sales Tax (10%)
```sql
SELECT 
  flavor_name,
  price,
  price * 1.10 as price_with_tax
FROM IceCream;
```

**Output:**
| flavor_name  | price | price_with_tax |
|--------------|-------|----------------|
| Vanilla      | 3.50  | 3.85           |
| Chocolate    | 3.50  | 3.85           |
| Strawberry   | 3.75  | 4.125          |
| Mint Chip    | 4.00  | 4.40           |
| Cookie Dough | 4.25  | 4.675          |
| Bubblegum    | 3.50  | 3.85           |
| Rocky Road   | 4.00  | 4.40           |

### Calculate Total Revenue
```sql
SELECT 
  flavor_name,
  scoops_sold,
  price,
  scoops_sold * price as total_revenue
FROM IceCream;
```

**Output:**
| flavor_name  | scoops_sold | price | total_revenue |
|--------------|-------------|-------|---------------|
| Vanilla      | 450         | 3.50  | 1575.00       |
| Chocolate    | 520         | 3.50  | 1820.00       |
| Strawberry   | 380         | 3.75  | 1425.00       |
| Mint Chip    | 290         | 4.00  | 1160.00       |
| Cookie Dough | 510         | 4.25  | 2167.50       |
| Bubblegum    | 220         | 3.50  | 770.00        |
| Rocky Road   | 340         | 4.00  | 1360.00       |

Wow! Cookie Dough made the most money!

## Using AS to Rename Columns

The `AS` keyword lets you give columns a new, friendlier name in your results:

```sql
SELECT 
  flavor_name AS Flavor,
  price AS Cost,
  scoops_sold AS Popularity
FROM IceCream;
```

**Output:**
| Flavor       | Cost | Popularity |
|--------------|------|------------|
| Vanilla      | 3.50 | 450        |
| Chocolate    | 3.50 | 520        |
| Strawberry   | 3.75 | 380        |
| Mint Chip    | 4.00 | 290        |
| Cookie Dough | 4.25 | 510        |
| Bubblegum    | 3.50 | 220        |
| Rocky Road   | 4.00 | 340        |

See how the column names changed? Much friendlier!

## Quick Tips

### Tip 1: SELECT Everything is Lazy (but OK for learning!)
```sql
SELECT * FROM Table;  -- Quick but shows everything
SELECT name, age FROM Table;  -- Better - only what you need!
```

### Tip 2: Calculations are Fun!
```sql
SELECT price * 2 as double_price FROM IceCream;
SELECT age + 10 as age_in_10_years FROM Customers;
```

### Tip 3: You Can Select Without a Table!
```sql
SELECT 5 + 5;  -- Shows: 10
SELECT 'Hello World!';  -- Shows: Hello World!
SELECT 2026 - 1996;  -- Shows: 30
```

This works in Programiz! Try it!

:::tip[Column Names Matter!]
When you select columns, spell them exactly right! SQL is case-sensitive in some databases. If you get an error saying "no such column", check your spelling!
:::
